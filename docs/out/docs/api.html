Performance Considerations

Caching

LPM caches repository data to minimize network requests:

  • Repository metadata: 5 minute TTL
  • VM definitions: Until repository update
  • Binary checksums: Permanent cache

Concurrency

LPM uses file locking to prevent concurrent operations:

Rate Limiting

When accessing external resources:

  • GitHub API: 60 requests/hour (unauthenticated)
  • GitHub API: 5000 requests/hour (authenticated)
  • Binary downloads: No built-in limit
