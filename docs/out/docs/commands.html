Commands

lpm install

Install a VM plugin from a repository.

Syntax

Flags

  • --vm string - Name of the VM to install (required)
  • --version string - Specific version to install (optional, defaults to latest)
  • --repository string - Repository alias to use (optional, defaults to core)
  • --force - Force installation even if already installed

Examples

Behavior

  1. Checks if VM is already installed
  2. Downloads repository if not cached
  3. Locates VM definition in repository
  4. Downloads VM binary from URL or builds from source
  5. Verifies checksums if provided
  6. Installs to plugin directory
  7. Updates state file

lpm uninstall

Remove an installed VM plugin.

Syntax

Flags

  • --vm string - Name of the VM to uninstall (required)
  • --force - Skip confirmation prompt

Examples

Behavior

  1. Checks if VM is installed
  2. Prompts for confirmation (unless --force)
  3. Removes VM binary from plugin directory
  4. Updates state file

lpm update

Update repository definitions to get latest VM and subnet configurations.

Syntax

Flags

  • --repository string - Specific repository to update (optional, updates all if omitted)

Examples

Behavior

  1. Performs git pull on specified repository(ies)
  2. Updates local cache of VM and subnet definitions
  3. Reports any new or updated definitions

lpm upgrade

Upgrade installed VM plugins to their latest versions.

Syntax

Flags

  • --vm string - Specific VM to upgrade (optional, upgrades all if omitted)
  • --force - Force upgrade even if already at latest version

Examples

Behavior

  1. Checks for newer versions of installed VMs
  2. Downloads and installs updates
  3. Preserves VM configuration
  4. Updates state file with new versions

lpm list-repositories

Display all configured plugin repositories.

Syntax

Flags

  • --verbose - Show detailed repository information
  • --json - Output in JSON format

Examples

Output

lpm add-repository

Add a new plugin repository.

Syntax

Flags

  • --alias string - Local name for the repository (required)
  • --url string - Git repository URL (required)
  • --branch string - Git branch to track (default: master)
  • --credentials-file string - Credentials for private repository

Examples

Behavior

  1. Validates repository URL
  2. Clones repository to local cache
  3. Adds to state file
  4. Repository becomes available for install commands

lpm remove-repository

Remove a configured plugin repository.

Syntax

Flags

  • --alias string - Repository alias to remove (required)
  • --force - Skip confirmation prompt

Examples

Behavior

  1. Checks if repository exists
  2. Ensures no installed VMs depend on it
  3. Prompts for confirmation (unless --force)
  4. Removes repository from cache and state

lpm join-subnet

Join a subnet by installing all required VM plugins.

Syntax

Flags

  • --subnet string - Name of subnet to join (required)
  • --repository string - Repository containing subnet definition
  • --node-config string - Custom node configuration file
  • --force - Force join even if some VMs already installed

Examples

Behavior

  1. Locates subnet definition in repository
  2. Identifies required VM plugins
  3. Installs all required VMs
  4. Configures node to track subnet
  5. Updates state file

Exit Codes

LPM uses standard exit codes:

  • 0 - Success
  • 1 - General error
  • 2 - Invalid arguments
  • 3 - Repository error
  • 4 - Network error
  • 5 - Permission error
  • 6 - State corruption
  • 7 - VM not found
  • 8 - Already installed
  • 9 - Node communication error

Tips and Best Practices

  1. Always backup state before major operations
  2. Use specific versions in production environments
  3. Keep repositories updated with regular lpm update
  4. Monitor disk space in plugin directory
  5. Use credentials files instead of inline passwords
  6. Test upgrades in development before production
  7. Document custom repositories in team wikis
  8. Use meaningful aliases for repositories
